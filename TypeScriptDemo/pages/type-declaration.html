Interfaces - Indexers

TypeScript allows for the definition of indexable types. This allows you to define dictionary-like interfaces. Typescript supports indexable types via numeric and string indexers. However, all indexers are ultimately resolved to strings. That means defining a number index with a value of 100 is actually stored in the created type as "100". If both a numeric and string indexer are specified on an interface, the compiler requires that the resulting return types are identical. Generics & Casting

Much like current Java and C# syntax, TypeScript also supports generics and type casting. This allows a developer to apply functionality to an inconsistent or evolving object structure. In this example, a simple drawing program is created by creating several tools that each have a "use()" method. The UI is then created such that a Button class which accepts a generic value in the constructor, is used to select which tool to work with on a canvas. When a button is pressed, the associated generic tool is activated by casting it to the appropriate interface and then using it. The use of generics here makes it very easy to create a command pattern for this drawing example. This feature is also useful when implementing factory patterns in an application.
