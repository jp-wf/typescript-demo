Necessary Tools

There are a number of routes you could go to get started working in TypeScript. Since it was originally developed by Microsoft this presentation will outline their tooling for getting up and going. The first basic tool you'll need is the IDE to work in, you can choose either Visual Studio 2015 Community Edition or Visual Studio Code. Both of these products are available from the Visual Studio Downloads page.

These products support TypeScript out of the box, but it is a good idea to make sure you have the latest version, which can be obtained from the following link: TypeScript for Visual Studio 2015. The development of TypeScript in the past few years has been very active and a lot of the features that were added in the 2.0 version really make working with the language much easier. That being the case, you'll want to make sure you running the latest stable version.

Other Recommended Tooling

The rise of package management systems has greatly eased intregration of external dependencies in projects in the past decade or so. In the javascript space, no system is more prominent than Node.js and NPM. To get the most out of TypeScript development, it is probably more of a requirement than a recommendation to install these products. Node.js is available for most major platforms. It comes with a simple version of NPM built-in, but the full version can be installed via the following command:

npm install npm -g

Once you have Node.js and NPM installed, you can leverage a lot of tooling to aid in TypeScript development. There are many module systems, build tools and testing tools, and Type Definitions for existing Javascript libraries the can be retrieved via NPM. Additionally, if you create a package.json file for your project using ‘npm init’ you can interact with packages using a dynamic user interface built into Visual Studio.

Other Recommended Tooling (Cont'd)

Having Node.js and NPM installed gives access to a plethora of development and testing tools.  A couple of examples are outlined below:



For task automation, a great tool is Gulp. This tool allows for file concatenation, packaging, linting, compiling sass, watching for file changes, minifying and uglifying code among many other tasks. Furthermore, Visual Studio has a window called the Task Runner Explorer, which allows for running and managing Gulp tasks directly through the IDE.



Jasmine is a behavior-driven testing framework that was developed for JavaScript. Since TypeScript transpiles down to JavaScript, it works equally well for testing your TypeScript code. The testing syntax can be supported by importing the @types/jasmine module from NPM. Jasmine is covered in more detail later in this presentation.
