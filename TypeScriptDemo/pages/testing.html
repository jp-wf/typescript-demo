How TypeScript Enables Unit Testing

The modular nature of TypeScript allows for ease of setup when configuring unit tests for an application. The object-oriented class layer and generic support enable solution designs where individuals components can be isolated and tested. Furthermore, this allows for simple mocking of dependencies and subsystems that would be otherwise hard to interact with. Since the TypeScript is transpiled, you have an extra layer of safety when developing your tests, just like you do when you're developing the application code itself.

An example of this modular nature is the PageViewer class that is driving this entire presentation. This class manages the loading of page content and the navigation between the various pages. It also reports state back to the UI for displaying current Page Titles and Subtitles, as well as allowed navigation from the current page. Writing Test Specs Using Jasmine

As mentioned earlier in the presentation, Jasmine is a JavaScript testing framework that works well for developing unit tests for front-end code. Since the type definition for Jasmine was imported, the test specs look almost identical to what they would look like in JavaScript. There are a couple exceptions such as the definition of functions using arrow notation and the import statements at the top of the test. The test spec is transpiled into JavaScript before it is run through the test runner, just like the application code.

Jasmine includes a standalone test runner, which is embedded on the next page of the presentation. This works well for visualization of test status while developing code and tests. For automated testing, there are a number of solutions that integrate with Jasmine. Most notable is the test runner developed by Google that was used for Angular. Karma allows for the running of tests from several testing frameworks. It supports both a visual mode and lights out operation. It can be integrated with task runnner tools such as Grunt and Gulp. It also generates common output formats that can provide build time status of unit tests including code coverage metrics. These formats can be loaded and included with build status in products like Jenkins.
